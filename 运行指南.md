# 游戏攻略知识库系统 - 完整运行指南

## 项目概述

这是一个完整的前后端分离项目，包含：
- **后端**: Spring Boot 3.x + MyBatis + Spring Security + JWT + PostgreSQL
- **前端**: Vue 3 + Vite + Element Plus + Pinia

## 功能特性

### 后端功能
- ✅ JWT 认证授权
- ✅ 用户注册登录
- ✅ 基于角色的权限控制（ADMIN/USER）
- ✅ 游戏管理 CRUD
- ✅ 攻略管理 CRUD + 搜索
- ✅ 分类管理 CRUD
- ✅ 标签管理 CRUD
- ✅ 统一返回结构
- ✅ 全局异常处理
- ✅ 分页查询

### 前端功能
- ✅ 用户登录注册
- ✅ JWT Token 自动携带
- ✅ 路由守卫
- ✅ 权限控制（ADMIN 可增删改，USER 只读）
- ✅ 游戏管理页面
- ✅ 攻略管理页面（支持搜索）
- ✅ 分类管理页面
- ✅ 标签管理页面
- ✅ Dashboard 统计页面

## 快速开始

### 一、数据库准备

#### 1. 安装 PostgreSQL 15

确保已安装 PostgreSQL 15 或更高版本。

#### 2. 创建数据库并初始化

```bash
# 登录 PostgreSQL
psql -U postgres

# 创建数据库
CREATE DATABASE game_guide;

# 退出
\q

# 执行初始化脚本
psql -U postgres -d game_guide -f database/init.sql
psql -U postgres -d game_guide -f database/user.sql
```

#### 3. 默认用户

系统已创建两个默认用户：

| 用户名 | 密码 | 角色 |
|--------|------|------|
| admin  | admin123 | ADMIN |
| user   | admin123 | USER |

### 二、后端启动

#### 方式一：使用 Maven 运行

```bash
# 进入后端目录
cd game-guide-backend

# 安装依赖并运行
mvn clean install
mvn spring-boot:run
```

#### 方式二：使用 IDE 运行

1. 使用 IntelliJ IDEA 或 Eclipse 打开 `game-guide-backend` 项目
2. 等待 Maven 依赖下载完成
3. 运行 `GameGuideApplication.java` 主类

#### 方式三：使用 Docker Compose

```bash
cd game-guide-backend
docker-compose up -d
```

**后端访问地址**: http://localhost:8080/api

### 三、前端启动

#### 1. 安装依赖

```bash
# 进入前端目录
cd frontend

# 安装依赖（使用 npm）
npm install

# 或使用 pnpm（推荐）
pnpm install

# 或使用 yarn
yarn install
```

#### 2. 启动开发服务器

```bash
npm run dev
```

**前端访问地址**: http://localhost:5173

### 四、测试登录

1. 打开浏览器访问 http://localhost:5173
2. 使用默认账号登录：
   - 管理员：`admin` / `admin123`
   - 普通用户：`user` / `admin123`

## API 接口说明

### 认证接口（无需 Token）

#### 用户注册
```
POST /api/auth/register
Content-Type: application/json

{
  "username": "testuser",
  "password": "123456"
}
```

#### 用户登录
```
POST /api/auth/login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin123"
}

响应：
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "username": "admin",
    "role": "ADMIN"
  }
}
```

### 业务接口（需要 Token）

所有业务接口需要在请求头中携带 Token：
```
Authorization: Bearer {token}
```

#### 权限说明
- **GET 请求**: 所有登录用户可访问
- **POST/PUT/DELETE 请求**: 仅 ADMIN 角色可访问

#### 游戏管理
```
GET    /api/games?pageNum=1&pageSize=10    # 分页查询
GET    /api/games/all                      # 查询所有
GET    /api/games/{id}                     # 查询详情
POST   /api/games                          # 创建（ADMIN）
PUT    /api/games/{id}                     # 更新（ADMIN）
DELETE /api/games/{id}                     # 删除（ADMIN）
```

#### 攻略管理
```
GET    /api/guides?pageNum=1&pageSize=10           # 分页查询
```

## 配置说明

### 后端配置 (application.yml)

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/game_guide
    username: postgres
    password: postgres

jwt:
  secret: game-guide-secret-key-for-jwt-token-generation-must-be-at-least-256-bits
  expiration: 86400000  # 24小时
```

### 前端配置 (vite.config.js)

```javascript
server: {
  port: 5173,
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true
    }
  }
}
```

## 常见问题

### 1. 后端启动失败

**问题**: 数据库连接失败
**解决**: 检查 PostgreSQL 是否启动，数据库配置是否正确

**问题**: 端口 8080 被占用
**解决**: 修改 `application.yml` 中的 `server.port`

### 2. 前端启动失败

**问题**: 依赖安装失败
**解决**: 删除 `node_modules` 和 `package-lock.json`，重新安装

**问题**: 端口 5173 被占用
**解决**: 修改 `vite.config.js` 中的 `server.port`

### 3. 登录后 Token 失效

**问题**: Token 过期
**解决**: 重新登录获取新 Token

### 4. 权限不足

**问题**: 普通用户无法执行增删改操作
**解决**: 使用 ADMIN 角色账号登录

## 生产部署

### 后端部署

```bash
# 打包
mvn clean package -DskipTests

# 运行
java -jar target/game-guide-backend-1.0.0.jar
```

### 前端部署

```bash
# 构建
npm run build

# 部署 dist 目录到 Nginx 或其他 Web 服务器
```

### Nginx 配置示例

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端
    location / {
        root /path/to/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端代理
    location /api {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```
