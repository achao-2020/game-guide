# 游戏攻略知识库系统 - 运行指南

## 项目概述

这是一个完整的前后端分离项目，包含：
- **后端**: Spring Boot 3.x + MyBatis + Spring Security + JWT + PostgreSQL
- **前端**: Vue 3 + Vite + Element Plus + Pinia

## 功能特性

### 后端功能
- ✅ JWT 认证授权
- ✅ 用户注册登录
- ✅ 基于角色的权限控制（ADMIN/USER）
- ✅ 游戏管理 CRUD
- ✅ 攻略管理 CRUD + 搜索
- ✅ 分类管理 CRUD
- ✅ 标签管理 CRUD
- ✅ 统一返回结构
- ✅ 全局异常处理
- ✅ 分页查询

### 前端功能
- ✅ 用户登录注册
- ✅ JWT Token 自动携带
- ✅ 路由守卫
- ✅ 权限控制（ADMIN 可增删改，USER 只读）
- ✅ 游戏管理页面
- ✅ 攻略管理页面（支持搜索）
- ✅ 分类管理页面
- ✅ 标签管理页面
- ✅ Dashboard 统计页面

## 一、本地部署

### 1. 数据库准备

#### 1.1 安装 PostgreSQL 15

确保已安装 PostgreSQL 15 或更高版本。

#### 1.2 创建数据库并初始化

```bash
# 登录 PostgreSQL
psql -U postgres

# 创建数据库
CREATE DATABASE game_guide;

# 退出
\q

# 执行初始化脚本
psql -U postgres -d game_guide -f database/init.sql
```

#### 1.3 默认用户

系统已创建两个默认用户：

| 用户名 | 密码 | 角色 |
|--------|------|------|
| admin  | admin123 | ADMIN |
| user   | admin123 | USER |

### 2. 后端启动

#### 2.1 使用 Maven 运行

```bash
# 进入后端目录
cd game-guide-backend

# 安装依赖并运行
mvn clean install
mvn spring-boot:run
```

#### 2.2 使用 IDE 运行

1. 使用 IntelliJ IDEA 或 Eclipse 打开 `game-guide-backend` 项目
2. 等待 Maven 依赖下载完成
3. 运行 `GameGuideApplication.java` 主类

**后端访问地址**: http://localhost:8080/api

### 3. 前端启动

#### 3.1 安装依赖

```bash
# 进入前端目录
cd frontend

# 安装依赖（使用 npm）
npm install

# 或使用 pnpm（推荐）
pnpm install

# 或使用 yarn
yarn install
```

#### 3.2 启动开发服务器

```bash
npm run dev
```

**前端访问地址**: http://localhost:5173

### 4. 测试登录

1. 打开浏览器访问 http://localhost:5173
2. 使用默认账号登录：
   - 管理员：`admin` / `admin123`
   - 普通用户：`user` / `admin123`

### 5. 配置说明

#### 后端配置 (application.yml)

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/game_guide
    username: postgres
    password: postgres

jwt:
  secret: game-guide-secret-key-for-jwt-token-generation-must-be-at-least-256-bits
  expiration: 86400000  # 24小时
```

#### 前端配置 (vite.config.js)

```javascript
server: {
  port: 5173,
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true
    }
  }
}
```

## 二、服务器 Docker 部署

### 1. 前置准备

#### 1.1 安装 Docker 和 Docker Compose

**Ubuntu/Debian:**
```bash
# 安装 Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 安装 Docker Compose
sudo apt install docker-compose

# 启动 Docker 服务
sudo systemctl start docker
sudo systemctl enable docker
```

**CentOS/RHEL:**
```bash
# 安装 Docker
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install docker-ce docker-ce-cli containerd.io

# 安装 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 启动 Docker 服务
sudo systemctl start docker
sudo systemctl enable docker
```

**验证安装:**
```bash
docker --version
docker-compose --version
```

### 2. 后端 Docker 部署

#### 2.1 准备 Dockerfile

在 `game-guide-backend` 目录创建 `Dockerfile`:

```dockerfile
FROM eclipse-temurin:17-jre

WORKDIR /app

# 复制已经构建好的 jar 文件
COPY target/*.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar"]
```

#### 2.2 本地构建 JAR 包

```bash
cd game-guide-backend

# 使用 Maven 构建
mvn clean package -DskipTests
```

#### 2.3 准备 docker-compose.yml

在 `game-guide-backend` 目录创建 `docker-compose.yml`:

```yaml
version: '3.8'

services:
  postgres:
    image: postgres:15-alpine
    container_name: game-guide-postgres
    environment:
      POSTGRES_DB: game_guide
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ../database/init.sql:/docker-entrypoint-initdb.d/init.sql
    networks:
      - game-guide-network
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5
    restart: unless-stopped

  backend:
    build:
      context: .
      dockerfile: Dockerfile
    container_name: game-guide-backend
    ports:
      - "8080:8080"
    environment:
      SPRING_DATASOURCE_URL: jdbc:postgresql://postgres:5432/game_guide
      SPRING_DATASOURCE_USERNAME: postgres
      SPRING_DATASOURCE_PASSWORD: postgres
    depends_on:
      postgres:
        condition: service_healthy
    networks:
      - game-guide-network
    restart: unless-stopped

volumes:
  postgres_data:

networks:
  game-guide-network:
    driver: bridge
```

#### 2.4 部署后端

```bash
cd game-guide-backend

# 启动所有服务（数据库 + 后端）
docker-compose up -d

# 查看运行状态
docker-compose ps

# 查看日志
docker-compose logs -f backend

# 停止所有服务
docker-compose down

# 停止并删除数据卷
docker-compose down -v
```

#### 2.5 常用管理命令

```bash
# 重启后端服务
docker-compose restart backend

# 重新构建并启动
docker-compose up -d --build

# 查看特定服务日志
docker-compose logs -f postgres
docker-compose logs -f backend

# 进入容器
docker-compose exec backend sh
docker-compose exec postgres psql -U postgres -d game_guide

# 查看容器资源使用情况
docker stats
```

### 3. 前端 Docker 部署

#### 3.1 准备 Dockerfile

在 `frontend` 目录创建 `Dockerfile`:

```dockerfile
# 构建阶段
FROM node:18-alpine AS builder

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .
RUN npm run build

# 生产阶段
FROM nginx:alpine

# 复制构建产物
COPY --from=builder /app/dist /usr/share/nginx/html

# 复制 Nginx 配置
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

#### 3.2 准备 nginx.conf

在 `frontend` 目录创建 `nginx.conf`:

```nginx
server {
    listen 80;
    server_name localhost;

    # 前端静态文件
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
        expires 1d;
        add_header Cache-Control "public, immutable";
    }

    # 后端 API 代理
    location /api {
        proxy_pass http://backend:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript 
               application/x-javascript application/xml+rss 
               application/json application/javascript;
}
```

#### 3.3 构建和运行前端

```bash
cd frontend

# 构建镜像
docker build -t game-guide-frontend .

# 运行容器（需要连接到后端网络）
docker run -d \
  --name game-guide-frontend \
  -p 80:80 \
  --network game-guide-backend_game-guide-network \
  game-guide-frontend

# 查看日志
docker logs -f game-guide-frontend

# 停止容器
docker stop game-guide-frontend

# 删除容器
docker rm game-guide-frontend
```

### 5. 生产环境配置建议

#### 5.1 修改默认密码

修改 `docker-compose.yml` 中的数据库密码：

```yaml
postgres:
  environment:
    POSTGRES_PASSWORD: your-secure-password

backend:
  environment:
    SPRING_DATASOURCE_PASSWORD: your-secure-password
```

#### 5.2 配置域名和 HTTPS

如果有域名，建议配置 HTTPS。修改 `frontend/nginx.conf`:

```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;

    # 其他配置保持不变...
}
```

#### 5.3 数据备份

```bash
# 备份数据库
docker-compose exec postgres pg_dump -U postgres game_guide > backup.sql

# 恢复数据库
docker-compose exec -T postgres psql -U postgres game_guide < backup.sql
```

#### 5.4 监控和日志

### 6. 防火墙配置

如果服务器启用了防火墙，需要开放端口：

```bash
# Ubuntu/Debian (UFW)
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw reload

# CentOS/RHEL (firewalld)
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --reload
```

## 常见问题

### 1. 后端启动失败

**问题**: 数据库连接失败
**解决**: 
- 检查 PostgreSQL 是否启动
- 检查数据库配置是否正确
- Docker 部署时确保容器在同一网络

**问题**: 端口 8080 被占用
**解决**: 修改 `application.yml` 中的 `server.port` 或 `docker-compose.yml` 中的端口映射

### 2. 前端启动失败

**问题**: 依赖安装失败
**解决**: 删除 `node_modules` 和 `package-lock.json`，重新安装

**问题**: 端口 5173 被占用
**解决**: 修改 `vite.config.js` 中的 `server.port`

### 3. Docker 相关问题

**问题**: Docker 镜像构建失败
**解决**: 
- 确保 Dockerfile 路径正确
- 检查网络连接
- 清理 Docker 缓存: `docker system prune -a`

**问题**: 容器无法访问
**解决**: 
- 检查容器是否运行: `docker-compose ps`
- 查看容器日志: `docker-compose logs`
- 检查防火墙设置

**问题**: 数据库初始化失败
**解决**: 
- 确保 `init.sql` 路径正确
- 删除数据卷重新创建: `docker-compose down -v && docker-compose up -d`

### 4. 登录后 Token 失效

**问题**: Token 过期
**解决**: 重新登录获取新 Token

### 5. 权限不足

**问题**: 普通用户无法执行增删改操作
**解决**: 使用 ADMIN 角色账号登录

## API 接口说明

### 认证接口（无需 Token）

#### 用户登录
```
POST /api/auth/login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin123"
}

响应：
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "username": "admin",
    "role": "ADMIN"
  }
}
```

### 业务接口（需要 Token）

所有业务接口需要在请求头中携带 Token：
```
Authorization: Bearer {token}
```

#### 权限说明
- **GET 请求**: 所有登录用户可访问
- **POST/PUT/DELETE 请求**: 仅 ADMIN 角色可访问

#### 游戏管理
```
GET    /api/games?pageNum=1&pageSize=10    # 分页查询
GET    /api/games/all                      # 查询所有
GET    /api/games/{id}                     # 查询详情
POST   /api/games                          # 创建（ADMIN）
PUT    /api/games/{id}                     # 更新（ADMIN）
DELETE /api/games/{id}                     # 删除（ADMIN）
```

#### 攻略管理
```
GET    /api/guides?pageNum=1&pageSize=10           # 分页查询
GET    /api/guides/search?keyword=xxx              # 搜索
GET    /api/guides/{id}                            # 查询详情
POST   /api/guides                                 # 创建（ADMIN）
PUT    /api/guides/{id}                            # 更新（ADMIN）
DELETE /api/guides/{id}                            # 删除（ADMIN）
```
